## 4. Запуск

- Запустить Docker на компьютере.
- Основные команды для запуска и работы с проектом через `make`:

  1. **Сборка проекта**  
     ```bash
     make build
     ```  
     Скачивает зависимости и собирает Docker-образы.

  2. **Поднять проект**  
     ```bash
     make up
     ```  
     Запускает контейнеры (PostgreSQL и приложение)

  3. **Остановить проект и удалить контейнеры**  
     ```bash
     make down
     ```  
     Останавливает контейнеры и очищает тома данных.

  4. **Сборка и запуск вместе**  
     ```bash
     make all
     ```  
     Запускает сразу build и up.

  5. **Прогон тестов**  
     ```bash
     make test
     ```  
     Запускает все тесты в папках `handlers`, `service` и `repository`.

# Отсебятина

## Добавленные коды ошибок

Добавил некоторые коды для ошибок:

- **INVALID_BODY** — используется при ошибках валидации входных данных.
- **TEAM_NOT_FOUND** — возвращается, когда запрашиваемая команда не существует.
- **INTERNAL_ERROR** — возникает при непредвиденных ошибках сервиса или базы данных.
- **TEAM_EXISTS** — возвращается, если команда с таким именем уже существует.

### Зачем это сделано

Эти коды позволяют точнее классифицировать ошибки, что в будущем может упростить обработку на стороне клиента и общие понимание возникшей проблемы.
